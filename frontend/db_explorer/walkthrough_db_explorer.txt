Vivorn Villa FCM - DB Explorer (Chrome/Web Edition)
--------------------------------------------------
Topic: Internal Database Inspection Tool (Web/Chrome Compatible)

1. Overview:
   To run on Chrome, the app cannot access local files directly (Browser Security).
   We have implemented a 'Backend Bridge' in the existing Node.js server to 
   allow the Flutter Web app to inspect the database.

2. Architecture:
   - Backend: Express router at `/api/dev/` provides SQL execution.
   - Frontend: Flutter Web uses `package:http` to communicate with the bridge.

3. Key Components:
   - Backend Controller: `backend/src/controllers/dev.controller.ts`
   - Backend Routes: `backend/src/routes/dev.routes.ts`
   - Frontend: `frontend/db_explorer/lib/main.dart`

4. How to Run:
   
   A. Ensure Backend is Running:
      $ cd backend
      $ npm run dev
   
   B. Run Flutter on Chrome:
      $ cd frontend/db_explorer
      $ flutter run -d chrome

5. Features:
   - Sidebar with Real-time Table List.
   - Dynamic Data Grid with Horizontal Scroll.
   - CRUD Operations (Add, Edit, Delete) mirrored to `fcm.db`.

Status: Chrome Support Ready (via Backend Bridge)
Developer Note: No Visual Studio needed for this mode.
